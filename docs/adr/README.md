# Architecture Decision Records (ADR)

Use this folder for decisions that should not be lost in chat or PR discussions.

## ADR Template

Use filenames:

1. `NNNN-short-title.md`

Suggested structure:

1. Title
2. Status (`proposed`, `accepted`, `superseded`)
3. Context
4. Decision
5. Consequences
6. Alternatives considered
7. Links to related PRs/issues/docs

## Initial ADR Candidates

1. Canonical docs strategy (`docs/` as source of truth).
2. Viewer modernization direction (toolkit-first custom viewer vs embedded alternatives).
3. LLM deterministic report settings and context boundaries.
